# `PreflightFailure` as exception, not value-result

`AgentRunner.run()`'s return type collapses from `AgentOutput | PreflightFailure` to `AgentOutput`. `PreflightFailure` becomes a `PycastleError` subclass — raised, not returned — joining `UsageLimitError` and `AgentTimeoutError` as the three typed exceptions that escape `AgentRunner.run()`. The merger (the only call site that runs with `skip_preflight=False` in production) becomes a `try/except PreflightFailure` block; every other formerly-defensive site that pattern-matched the union arm is deleted as dead code.

The trigger was issue #590: `AgentRunner.run()` had two parallel error-reporting mechanisms. `UsageLimitError` and `AgentTimeoutError` propagated as exceptions; `PreflightFailure` propagated as a value in the return type. Every caller therefore had to remember both mechanisms. Worse, the value-arm was almost entirely unused: every call site except the merger passed `skip_preflight=True`, which made `PreflightFailure` unreachable from those paths — but the type system still required `isinstance` discrimination, producing five sites of dead defensive code (`isinstance(result, PreflightFailure): return result` in `run_issue`, `assert not isinstance(...)` in `improve_phase`, `RuntimeError("...returned PreflightFailure unexpectedly")` in `planning` and `preflight-issue`, plus the `case PreflightFailure(failures=fs):` pretty-printer in `run_iteration` consuming an `errors` list that no production path ever populates with one). Sibling decision to ADR 0008: that ADR centralized exception handling for `UsageLimitError`; this one extends the same shape — control-flow escapes from `AgentRunner.run()` are exceptions — to the only remaining outlier.

## Considered Options

- **Status quo (PreflightFailure as a returned value).** Rejected: this is the failure mode. The dual mechanism forced every `AgentRunner.run()` caller to handle errors through both exception handling and isinstance discrimination, and the discrimination was almost entirely vestigial — five out of six dead.
- **Rename to `PreflightResult` and document the asymmetry as intentional (issue #590 Option B).** Rejected: the "structured inspectable result" justification doesn't hold up — `UsageLimitError` already carries inspectable state (`reset_time`) and proves exceptions can be inspectable. The asymmetry was incidental to the historical implementation order, not a load-bearing design choice. Renaming preserves the dual mechanism (callers still inspect the return type for one error class while catching others), and the rename itself is churn for no contract simplification.
- **New `AgentRunError` umbrella exception with `PreflightFailure(AgentRunError)`.** Rejected: speculative grouping. There is no `AgentRunError` today; `UsageLimitError` and `AgentTimeoutError` already escape `AgentRunner.run()` and are not grouped under such an umbrella. Introducing one purely to give `PreflightFailure` a more specific parent than `PycastleError` invents structure ahead of need. Cheap to add later if the umbrella gains a second subclass.
- **Rename to `PreflightFailureError` to fit the local `*Error` naming convention.** Rejected: "FailureError" reads awkwardly, the rename would touch 22 files for cosmetic alignment, and stdlib has unsuffixed exceptions (`KeyboardInterrupt`, `StopIteration`). The clarity gained from the suffix is not worth the churn.
- **Strict refactor (convert to exception; leave dead defensive code in place).** Rejected: the dead `isinstance` checks, asserts, and `RuntimeError` raises only existed because of the union return type. Once the union collapses, those guards have nothing to guard against. Leaving them as paranoia rots: the next reader can't tell whether a defensive check is shielding an active case or stale residue. Letting an unexpected `PreflightFailure` propagate as an uncaught exception is a louder, more honest signal than a silent `RuntimeError` rebrand.
- **Add `except PreflightFailure: raise` to the `runner.work()` retry loop in `AgentRunner.run()` to make the typed-propagate allowlist actually enforced.** Rejected for now: speculative. The raise site is at `agent_runner.py:156`, **before** the retry loop, so fail-soft physically cannot observe a `PreflightFailure` today. CONTEXT.md's listing of `PreflightFailure` in the allowlist is forward-looking — it becomes load-bearing only when a refactor moves preflight inside `runner.work()` (e.g., #594's lazy preflight on save-sha). Adding the explicit catch now means the next reader sees `except PreflightFailure: raise` in the retry loop and looks fruitlessly for what raises it from `runner.work()`. The catch belongs with the refactor that creates the need.
- **Convert to exception, delete the dead defensive sites, drop tests that exercise unreachable paths, document the rationale in an ADR.** The chosen design.

## Consequences

- `PreflightFailure` moves from `agent_result.py` to inherit from `PycastleError` (defined in `errors.py`); the dataclass field becomes the exception's first arg, accessed as `.failures: tuple[tuple[str,str,str], ...]`.
- `AgentRunner.run()`'s return type annotation collapses to `AgentOutput`. The protocol method in `AgentRunnerProtocol` and the test fake in `iteration/_deps.py` follow.
- The only live consumer — `merge_phase` at `iteration/merge.py:144-171` — converts to `try/except PreflightFailure: ...`. Behaviour is preserved bit-for-bit: print "Merge-time preflight failed; skipping conflict branch merge.", close the row, push if `auto_push`, return `MergeResult(clean=clean_issues, conflicts=conflict_issues)`. The "merge-time preflight skip" term in CONTEXT.md remains accurate.
- Five dead sites are deleted: the `isinstance(result, PreflightFailure): return result` in `run_issue` (`implement.py:123`), the `assert not isinstance(...)` in `improve_phase` (`improve.py:207`), and the `RuntimeError("...returned a PreflightFailure unexpectedly")` raises in `planning_phase` (`planning.py:61`) and `handle_preflight_failure` (`preflight.py:93`). The `case PreflightFailure(failures=fs):` arm in `run_iteration`'s error pretty-printer (`__init__.py:160-176`) is removed; `errors` aggregation in `implement_phase` (`implement.py:230`) simplifies from `list[tuple[dict, Exception | PreflightFailure]]` to `list[tuple[dict, Exception]]`. `_deps.py`'s logger `Protocol` follows.
- Tests that injected `PreflightFailure` into call sites that pass `skip_preflight=True` in production are deleted: `test_implement.py:179, 228, 695, 993` (implement-agent), `test_planning.py:127` (planner), `test_preflight.py:297` (preflight-issue reporter), and any related fakes in `test_orchestrator.py` / `test_deps.py` that built fixtures around them. Surviving tests adapt mechanically: `test_agent_result.py` constructs the exception and inspects `.failures`; `test_agent_runner.py:71` switches to `pytest.raises(PreflightFailure)`; `test_merge.py` switches its fake from a returned value to a raised exception, asserting on the same observable `MergeResult`.
- The fail-soft typed-propagate allowlist mention of `PreflightFailure` in CONTEXT.md (lines 251, 327) becomes accurate (the listed types are now all real exceptions) but remains nominally enforced: the raise site is before the retry loop, so the allowlist's exclusion of `PreflightFailure` from fail-soft is correct-by-construction rather than correct-by-explicit-catch. A future refactor that runs preflight inside `runner.work()` (e.g., #594's lazy preflight on save-sha) must add an explicit `except PreflightFailure: raise` clause to keep the exclusion enforced.
- Aligns `AgentRunner.run()`'s contract: every typed escape path is now an exception. Future maintainers adding a new failure mode have a single precedent — raise a `PycastleError` subclass — instead of choosing between two mechanisms.
- Issue #594 (lazy preflight) is unaffected by direction but reinforced: with preflight running at every save-sha consumer, an exception that bubbles to a single catch site is cheaper than per-consumer isinstance checks.
