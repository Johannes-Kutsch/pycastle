# Setup phase failures route upstream through auto_file_issue

Setup phase failures are handled operator-actionable failures, not preflight check failures and not unhandled crashes. When container startup, git identity propagation, gitdir/volume wiring, or dependency installation fails before role work or preflight checks begin, `run_iteration()` returns a setup-specific abort outcome: pycastle prints the local error, exits non-zero, and routes an upstream pycastle report through `auto_file_issue`, using API filing when `auto_file_bugs` is enabled and the prefilled URL fallback otherwise.

This is deliberately separate from two nearby paths. `PREFLIGHT_CHECKS` failures still go through the preflight-issue agent into the consuming project's tracker, because they are diagnosable check failures after the container baseline exists. `OperatorActionableGitError` still files on the consuming project's `origin`, because remote git reachability is an operator/environment issue rather than a pycastle defect. Setup phase failures sit between those categories: they are locally actionable enough to abort clearly, but upstream-relevant enough to use the pycastle bug-report path instead of the consuming repo.
