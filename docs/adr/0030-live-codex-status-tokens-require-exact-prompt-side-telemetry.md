# Live Codex status tokens require exact prompt-side telemetry

The status-row token column now has one cross-service meaning: the exact current prompt-side/context-window token count plus the session peak, never total turn spend and never an estimate. Claude already exposes that during assistant turns; Codex support is accepted only when the CLI `--json` stream exposes an exact prompt-side field early enough to update the live row. Pycastle will pin Codex to the first CLI version that provides that field, consume it at the `AgentService` seam, keep the last exact value visible while the stream is temporarily silent, and reject total-spend-style usage fields as a substitute because they would make the status row mean different things across services. Once that pinned Codex contract is in place, absence of the promised field is a Codex protocol error, not a silent fallback to end-of-turn behavior.
