# Architecture Decision Records

- [0001 â€” Runtime dependency installation](0001-runtime-dependency-installation.md): Install dev tools at container startup via `pip install -e '.[dev]'`, not baked into the image, so tools always match the current `pyproject.toml`.
- [0002 â€” CLI-native model shorthand resolution](0002-cli-native-model-shorthand-resolution.md): Pass model shorthands (e.g. `sonnet`) directly to the selected service CLI while validating non-empty shorthands at `pycastle run` startup, keeping `load_config` pure and testable.
- [0003 â€” Global user-scope config and env](0003-global-user-scope-config-and-env.md): Layer `defaults â†’ global (~/.config/pycastle/) â†’ local (pycastle/)` for `config.py` and `.env`, so credentials and preferences can be shared across projects without per-project re-entry.
- [0004 â€” GitHub PAT as sole credential](0004-github-pat-as-sole-credential.md): Route all GitHub access through `GithubService` with a single `GH_TOKEN` PAT, eliminating the dual `gh`-CLI / PAT auth paths that could fail independently.
- [0005 â€” Dual Claude account failover](0005-dual-claude-account-failover.md): Maintain an in-memory `AccountPool` of OAuth tokens; failover to a secondary account on usage-limit rather than sleeping, collapsing all auth onto `CLAUDE_CODE_OAUTH_TOKEN` vars.
- [0006 â€” Session resume across container teardown](0006-session-resume-across-container-teardown.md): Resume interrupted agents via `--resume <SESSION_UUID>` (UUID derived from role + worktree path) so in-progress reasoning survives usage-limit pool sleeps; non-typed Resume failures retry once then route to Failure-Report (amended #640).
- [0007 â€” Stage completion via role-session-dir state](0007-stage-completion-via-role-dir-not-commit-prefix.md): Signal implement/review stage completion by the presence of an empty `.pycastle-session/<role>/` dir (content cleared on success), not by `RALPH:` commit-prefix scan, eliminating the unrecoverable retry loop from issue #514.
- [0008 â€” Centralized UsageLimitError catch at iteration boundary](0008-centralized-usagelimit-catch-at-iteration-boundary.md): Catch `UsageLimitError` once at the top of `run_iteration` rather than in each phase's dispatcher arm, preventing silent crashes when a new phase is added without boilerplate.
- [0009 â€” PromptRenderer with scoped placeholder tiers](0009-prompt-renderer-with-scoped-placeholder-tiers.md): A single `PromptRenderer` validates all prompt templates at startup against a two-tier placeholder system (global pool + per-scope dynamic set), so prompt edits never require test changes.
- [0010 â€” Per-phase session namespacing for improve mode](0010-per-phase-session-namespacing-for-improve-mode.md): Split improve phase 03 (issue-filing) onto its own Claude session namespace (`issues`) away from phases 01/02/04 (`main`), injecting the PRD body via host-side prompt args rather than inheriting the full prior transcript.
- [0011 â€” Recently-closed filter for GitHub list endpoint](0011-recently-closed-filter-for-github-list-endpoint.md): `GithubService` tracks a self-healing `_recently_closed` set so `get_open_issues` filters out just-closed issues during the eventual-consistency window, preventing the planner from re-picking issues the Merger just closed.
- [0012 â€” Prompt rendering in AgentRunner, not ContainerRunner](0012-prompt-rendering-in-agent-runner-not-container-runner.md): Move all prompt-shape resolution from `ContainerRunner.work()` into `AgentRunner._build_prompt()`, decoupling the container runner from `PromptRenderer` and improve-mode session semantics.
- [0013 â€” Content-addressed PreflightCache; remove skip_preflight](0013-remove-skip-preflight-route-merger-through-cache.md): Replace iteration-scoped `Deps.preflight_verdict` with a single-slot cross-iteration `PreflightCache`; remove `RunRequest.skip_preflight`, `runner.preflight()`, and `PreflightFailure`; route the Merger's post-merge check through `get_safe_sha()`.
- [0014 â€” Pin implement-phase SHA from planner](0014-pin-implement-sha-from-planner.md): Reintroduce `PlanReady.sha`; `run_issue` consumes the planner's SHA directly and no longer calls `get_safe_sha`, closing the duplicate-issue race in #679 when HEAD advances between planning and implement.
- [0015 â€” Pluggable agent services with per-stage fallback](0015-pluggable-agent-services-per-stage-fallback.md): Introduce an `AgentService` abstraction at the streaming-execution seam; services own their own command construction, env injection, wire-format parsing, resume state, and account pooling. Each stage names a preferred service plus an optional `(service, model, effort)` fallback; cross-service handoff mid-stage works via WIP commits squashed at stage end.
- [0016 â€” Three-mode implement slice with explicit gate](0016-three-mode-implement-slice-with-explicit-gate.md): Split implement work into three mutually-exclusive slice modes (`refactor-slice`, `behavior-slice`, `docs-slice`) marked by GitHub labels and dispatched to three separate implement prompts; behavior prompt is two-phase, gated by mandatory `<first_behavior>` and `<failing_test>` tags carrying real pytest output before any non-test edit.
- [0017 â€” Semantic XML tags for agent prompts](0017-semantic-xml-tags-for-agent-prompts.md): All agent prompts use a fixed four-tag vocabulary (`<task>`, `<context>`, `<workflow>`, `<output>`) for top-level structure; standards fragments self-wrap with named tags (`<design-standards>`, `<implementation-standards>`, `<output-rules>`); `##` sub-headings stay as markdown inside their section.
- [0018 â€” Persist agent draft files inside the role session dir](0018-persist-agent-drafts-in-role-session-dir.md): Improve-mode prompts write `gh issue create --body-file` drafts to `<worktree>/.pycastle-session/improve/drafts/` instead of `/tmp/`, so drafts survive `UsageLimitError` container teardown via the role-session-dir preservation rule and unfiled slices are not silently lost on Claude `--resume`.
- [0019 â€” Implicit Docker image verify on `pycastle run`](0019-implicit-image-verify-on-pycastle-run.md): `pycastle run` invokes the same build path as `pycastle build` before starting containers, relying on Docker's layer cache to make the no-op case cheap, so stale images can't silently power a run after Dockerfile edits.
- [0020 â€” Codex authentication via per-role workspace-local `auth.json`](0020-codex-auth-via-per-role-bind-mount.md): `CodexService` authenticates each role's container through a per-role workspace-local copy of the host's ChatGPT-OAuth `auth.json`, seeded by `pycastle init` and lazily at fresh runtime state creation; per-role lineages diverge after first refresh, while Codex's server-generated `thread_id` is captured, persisted, and recovered from rollout files so `codex exec resume` never receives a pycastle-generated UUID.
- [0021 â€” Shared `gh` issue-tracker fragment in default prompts](0021-shared-gh-issue-tracker-fragment-in-default-prompts.md): Factor the GitHub issue-tracker recipe out of five prompts into a single shared include `_issue-tracker.md` injected via the `{{ISSUE_TRACKER}}` global placeholder, with `gh` CLI recipes (authenticating via `$GH_TOKEN`) installed in both agent Dockerfiles; eliminates the five-way copy-paste that produced the 2026-05-18 slice-agent hallucination and keeps tool-call token cost low.
- [0022 â€” Destination-scoped auto-issue gate, default off](0022-destination-scoped-auto-issue-gate.md): `auto_file_bugs` gates every auto-filed issue whose destination is the upstream pycastle repo (regardless of trigger severity) and defaults to `False`; a single `auto_file_issue` helper consolidates gate check, token resolution, API attempt, and prefilled-URL fallback so adding a new auto-report site (e.g. #807 usage-limit parse failures) is a one-line call.
- [0023 â€” Three-bucket Claude API error handling](0023-three-bucket-claude-api-error-handling.md): Classify every `is_error: true` Claude CLI result envelope into one of three buckets â€” 429 keeps existing `UsageLimitError` path, 5xx raises a transient-error exception (preserve worktree, cancel not-yet-started siblings, retry next iteration), 4xx-non-429 raises a hard-error exception except for the exact subscription-access denial handled by ADR 0032 (preserve worktree, auto-file via ADR 0022, halt orchestrator after current iteration); closes the silent-success-on-API-error window that caused #831.
- [0024 â€” Per-agent Claude CLI flag profiles](0024-per-agent-cli-flag-profiles.md): Hardcode a per-`AgentRole` flag profile in `claude_service.build_command` â€” Planner and divergence-resolver run `--bare`; Reviewer/Implementer/Merger keep full tools; `preflight-issue`/`improve scan` get `--disallowedTools "Edit Write NotebookEdit"`; every role gets `--disable-slash-commands`, `--exclude-dynamic-system-prompt-sections`, and (non-bare) `--strict-mcp-config --mcp-config '{}'`. Coupled with a divergence-resolver contract change: no CHECKS run inside the resolver, broken merges land on `main` and surface via the existing `get_safe_sha` preflight pass.
- [0025 â€” Unified status-row wrapper](0025-unified-status-row-wrapper.md): Collapse `phase_row` / `agent_row` into a single `status_row` async CM parameterised on `kind`, `must_close`, and `color_key`; apply phase-row exception handling uniformly so agent rows paint `"interrupted"` on `UsageLimitError` / `AgentTimeoutError`; replace caller-name `#N` regex for palette-index lookup with an explicit `color_key: int | None` kwarg. Unblocks Preflight's `"finished, all tests green"` verdict via the unified `.close()` mechanism.
- [0026 â€” Operator-actionable git failures retry-then-escalate to the consuming project](0026-operator-actionable-git-failures.md): Remote-touching git ops (`pull`, `fetch`, `push`) retry on `Permission denied` / auth / network with `[10s, 60s, 300s]` backoff (4 attempts); on retry exhaustion (or stable misconfig like `repository not found`) the orchestrator files a `bug + needs-triage` issue on the **consuming project's `origin`** â€” never the pycastle tracker â€” deduped cross-process via `gh issue list`, then exits non-zero. Closes the #875/#876 class where transient GitHub SSH blips crashed pycastle with a stack trace and auto-filed environmental issues on the pycastle bug tracker.
- [0027 â€” `max_parallel` is a concurrency cap, not a per-iteration batch cap](0027-max-parallel-as-concurrency-cap.md): `max_parallel` limits concurrent agents only; every planned issue runs in its iteration, gated by two coordinating semaphores (`max_parallel` for agents, `max_parallel + 1` for worktrees), and a single merge phase closes the iteration regardless of value.
- [0028 â€” Discipline-forcing artifact tags for behavior-slice implement and review](0028-discipline-forcing-artifact-tags.md): Every required step in a prompt either emits a load-bearing artifact tag the agent can't fake without doing the work, or is acknowledged as best-effort. Collapses `implement/behavior.md`'s Phase A/B into a per-behavior loop requiring a repeatable `<behavior>` tag (name + observable surface + test file path + failing-test pytest paste); adds `<reviewed_diff>` and `<checks_passed>` to the Reviewer. Retires `<first_behavior>` / `<failing_test>`.
- [0029 â€” Per-service Docker images](0029-per-service-docker-images.md): Superseded by ADR 0034. Previously split `pycastle build` into per-service images (`Dockerfile.claude`, `Dockerfile.codex`) tagged `<base>-<service>`, each containing only what its service needs; runtime selected image per agent role from `StageOverride.service`.
- [0030 â€” Preserve transient worktree for Failure-Report agent](0030-preserve-transient-worktree-for-failure-analysis.md): `transient_worktree` preserves the worktree on `AgentFailedError` so the Failure-Report agent can mount it; handler cleans up after the agent completes.
- [0031 â€” Pin agent CLI versions and fix codex resume sandbox rejection](0031-pin-agent-cli-versions-and-fix-codex-resume-sandbox.md): Emit `--sandbox` only on fresh Codex runs and pin both agent CLIs to tested versions so resume paths and Docker builds stay stable.
- [0032 â€” Claude subscription denial is permanent account exhaustion](0032-claude-subscription-denial-is-permanent-account-exhaustion.md): Treat the exact Claude subscription-access denial as permanent exhaustion of the active Claude account, not as a generic 403 or an Anthropic API-key fallback.
- [0033 — Live Codex status tokens require exact prompt-side telemetry](0033-live-codex-status-tokens-require-exact-prompt-side-telemetry.md): Give the status-row token column one cross-service meaning — exact prompt-side/context-window tokens plus session peak — and only enable live Codex updates when the pinned Codex CLI exposes an exact prompt-side field on the `--json` stream early enough to drive the row.
- [0034 — Universal agent image and stage priority chains](0034-universal-agent-image-and-stage-priority-chains.md): Keep nested `StageOverride.fallback` syntax but interpret chains as ordered priority lists with unresolved services skipped, and replace per-service Docker images with one universal image containing all supported service CLIs.
- [0035 — Host-owned conflict merge commits](0035-host-owned-conflict-merge-commits.md): Merger resolves one host-started conflict merge at a time under the same trust boundary as Implementer/Reviewer; pycastle creates the merge commit and validates branch ancestry before closing the issue.
