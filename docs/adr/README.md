# Architecture Decision Records

- [0001 ‚Äî Runtime dependency installation](0001-runtime-dependency-installation.md): Install dev tools at container startup via `pip install -e '.[dev]'`, not baked into the image, so tools always match the current `pyproject.toml`.
- [0002 ‚Äî CLI-native model shorthand resolution](0002-cli-native-model-shorthand-resolution.md): Pass model shorthands (e.g. `sonnet`) directly to the Claude CLI rather than resolving them at config load time, keeping `load_config` pure and testable.
- [0003 ‚Äî Global user-scope config and env](0003-global-user-scope-config-and-env.md): Layer `defaults ‚Üí global (~/.config/pycastle/) ‚Üí local (pycastle/)` for `config.py` and `.env`, so credentials and preferences can be shared across projects without per-project re-entry.
- [0004 ‚Äî GitHub PAT as sole credential](0004-github-pat-as-sole-credential.md): Route all GitHub access through `GithubService` with a single `GH_TOKEN` PAT, eliminating the dual `gh`-CLI / PAT auth paths that could fail independently.
- [0005 ‚Äî Dual Claude account failover](0005-dual-claude-account-failover.md): Maintain an in-memory `AccountPool` of OAuth tokens; failover to a secondary account on usage-limit rather than sleeping, collapsing all auth onto `CLAUDE_CODE_OAUTH_TOKEN` vars.
- [0006 ‚Äî Session resume across container teardown](0006-session-resume-across-container-teardown.md): Resume interrupted agents via `--resume <SESSION_UUID>` (UUID derived from role + worktree path) so in-progress reasoning survives usage-limit pool sleeps; non-typed Resume failures retry once then route to Failure-Analysis (amended #640).
- [0007 ‚Äî Stage completion via role-session-dir state](0007-stage-completion-via-role-dir-not-commit-prefix.md): Signal implement/review stage completion by the presence of an empty `.pycastle-session/<role>/` dir (content cleared on success), not by `RALPH:` commit-prefix scan, eliminating the unrecoverable retry loop from issue #514.
- [0008 ‚Äî Centralized UsageLimitError catch at iteration boundary](0008-centralized-usagelimit-catch-at-iteration-boundary.md): Catch `UsageLimitError` once at the top of `run_iteration` rather than in each phase's dispatcher arm, preventing silent crashes when a new phase is added without boilerplate.
- [0009 ‚Äî PromptRenderer with scoped placeholder tiers](0009-prompt-renderer-with-scoped-placeholder-tiers.md): A single `PromptRenderer` validates all prompt templates at startup against a two-tier placeholder system (global pool + per-scope dynamic set), so prompt edits never require test changes.
- [0010 ‚Äî Per-phase session namespacing for improve mode](0010-per-phase-session-namespacing-for-improve-mode.md): Split improve phase 03 (issue-filing) onto its own Claude session namespace (`issues`) away from phases 01/02/04 (`main`), injecting the PRD body via host-side prompt args rather than inheriting the full prior transcript.
- [0011 ‚Äî Recently-closed filter for GitHub list endpoint](0011-recently-closed-filter-for-github-list-endpoint.md): `GithubService` tracks a self-healing `_recently_closed` set so `get_open_issues` filters out just-closed issues during the eventual-consistency window, preventing the planner from re-picking issues the Merger just closed.
- [0012 ‚Äî Prompt rendering in AgentRunner, not ContainerRunner](0012-prompt-rendering-in-agent-runner-not-container-runner.md): Move all prompt-shape resolution from `ContainerRunner.work()` into `AgentRunner._build_prompt()`, decoupling the container runner from `PromptRenderer` and improve-mode session semantics.
- [0013 ‚Äî Content-addressed PreflightCache; remove skip_preflight](0013-remove-skip-preflight-route-merger-through-cache.md): Replace iteration-scoped `Deps.preflight_verdict` with a single-slot cross-iteration `PreflightCache`; remove `RunRequest.skip_preflight`, `runner.preflight()`, and `PreflightFailure`; route the Merger's post-merge check through `get_safe_sha()`.
- [0014 ‚Äî Pin implement-phase SHA from planner](0014-pin-implement-sha-from-planner.md): Reintroduce `PlanReady.sha`; `run_issue` consumes the planner's SHA directly and no longer calls `get_safe_sha`, closing the duplicate-issue race in #679 when HEAD advances between planning and implement.
- [0015 ‚Äî Pluggable agent services with per-stage fallback](0015-pluggable-agent-services-per-stage-fallback.md): Introduce an `AgentService` abstraction at the streaming-execution seam; services own their own command construction, env injection, wire-format parsing, resume state, and account pooling. Each stage names a preferred service plus an optional `(service, model, effort)` fallback; cross-service handoff mid-stage works via WIP commits squashed at stage end.
- [0016 ‚Äî Three-mode implement slice with explicit gate](0016-three-mode-implement-slice-with-explicit-gate.md): Split implement work into three mutually-exclusive slice modes (`refactor-slice`, `behavior-slice`, `docs-slice`) marked by GitHub labels and dispatched to three separate implement prompts; behavior prompt is two-phase, gated by mandatory `<first_behavior>` and `<failing_test>` tags carrying real pytest output before any non-test edit.
- [0017 ‚Äî Semantic XML tags for agent prompts](0017-semantic-xml-tags-for-agent-prompts.md): All agent prompts use a fixed four-tag vocabulary (`<task>`, `<context>`, `<workflow>`, `<output>`) for top-level structure; standards fragments self-wrap with named tags (`<design-standards>`, `<implementation-standards>`, `<output-rules>`); `##` sub-headings stay as markdown inside their section.
- [0018 ‚Äî Persist agent draft files inside the role session dir](0018-persist-agent-drafts-in-role-session-dir.md): Improve-mode prompts write `gh issue create --body-file` drafts to `<worktree>/.pycastle-session/improve/drafts/` instead of `/tmp/`, so drafts survive `UsageLimitError` container teardown via the role-session-dir preservation rule and unfiled slices are not silently lost on Claude `--resume`.
- [0019 ‚Äî Implicit Docker image verify on `pycastle run`](0019-implicit-image-verify-on-pycastle-run.md): `pycastle run` invokes the same build path as `pycastle build` before starting containers, relying on Docker's layer cache to make the no-op case cheap, so stale images can't silently power a run after Dockerfile edits.
- [0020 ‚Äî Codex authentication via per-role bind-mount of `~/.codex/auth.json`](0020-codex-auth-via-per-role-bind-mount.md): `CodexService` authenticates each role's container by bind-mounting a per-role copy of the host's ChatGPT-OAuth `auth.json`, seeded eagerly at `pycastle init`; per-role lineages diverge after first refresh, which avoids concurrent-refresh races at the cost of requiring re-init after `codex login` to add new roles.
- [0021 ‚Äî Shared `gh` issue-tracker fragment in default prompts](0021-shared-gh-issue-tracker-fragment-in-default-prompts.md): Factor the GitHub issue-tracker recipe out of five prompts into a single shared include `_issue-tracker.md` injected via the `{{ISSUE_TRACKER}}` global placeholder, with `gh` CLI recipes (authenticating via `$GH_TOKEN`) installed in both agent Dockerfiles; eliminates the five-way copy-paste that produced the 2026-05-18 slice-agent hallucination and keeps tool-call token cost low.
- [0022 ‚Äî Destination-scoped auto-issue gate, default off](0022-destination-scoped-auto-issue-gate.md): `auto_file_bugs` gates every auto-filed issue whose destination is the upstream pycastle repo (regardless of trigger severity) and defaults to `False`; a single `auto_file_issue` helper consolidates gate check, token resolution, API attempt, and prefilled-URL fallback so adding a new auto-report site (e.g. #807 usage-limit parse failures) is a one-line call.
- [0023 ‚Äî Three-bucket Claude API error handling](0023-three-bucket-claude-api-error-handling.md): Classify every `is_error: true` Claude CLI result envelope into one of three buckets ‚Äî 429 keeps existing `UsageLimitError` path, 5xx raises a transient-error exception (preserve worktree, cancel not-yet-started siblings, retry next iteration), 4xx-non-429 raises a hard-error exception (preserve worktree, auto-file via ADR 0022, halt orchestrator after current iteration); closes the silent-success-on-API-error window that caused #831.
- [0024 ‚Äî Per-agent Claude CLI flag profiles](0024-per-agent-cli-flag-profiles.md): Hardcode a per-`AgentRole` flag profile in `claude_service.build_command` ‚Äî Planner and divergence-resolver run `--bare`; Reviewer/Implementer/Merger keep full tools; `preflight-issue`/`improve scan` get `--disallowedTools "Edit Write NotebookEdit"`; every role gets `--disable-slash-commands`, `--exclude-dynamic-system-prompt-sections`, and (non-bare) `--strict-mcp-config --mcp-config '{}'`. Coupled with a divergence-resolver contract change: no CHECKS run inside the resolver, broken merges land on `main` and surface via the existing `get_safe_sha` preflight pass.
- [0025 ‚Äî Unified status-row wrapper](0025-unified-status-row-wrapper.md): Collapse `phase_row` / `agent_row` into a single `status_row` async CM parameterised on `kind`, `must_close`, and `color_key`; apply phase-row exception handling uniformly so agent rows paint `"interrupted"` on `UsageLimitError` / `AgentTimeoutError`; replace caller-name `#N` regex for palette-index lookup with an explicit `color_key: int | None` kwarg. Unblocks Preflight's `"finished, all tests green"` verdict via the unified `.close()` mechanism.
- [0026 ‚Äî Operator-actionable git failures retry-then-escalate to the consuming project](0026-operator-actionable-git-failures.md): Remote-touching git ops (`pull`, `fetch`, `push`) retry on `Permission denied` / auth / network with `[10s, 60s, 300s]` backoff (4 attempts); on retry exhaustion (or stable misconfig like `repository not found`) the orchestrator files a `bug + needs-triage` issue on the **consuming project's `origin`** ‚Äî never the pycastle tracker ‚Äî deduped cross-process via `gh issue list`, then exits non-zero. Closes the #875/#876 class where transient GitHub SSH blips crashed pycastle with a stack trace and auto-filed environmental issues on the pycastle bug tracker.
