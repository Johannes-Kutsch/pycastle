# Architecture Decision Records

- [0001 — Runtime dependency installation](0001-runtime-dependency-installation.md): Install dev tools at container startup via `pip install -e '.[dev]'`, not baked into the image, so tools always match the current `pyproject.toml`.
- [0002 — CLI-native model shorthand resolution](0002-cli-native-model-shorthand-resolution.md): Pass model shorthands (e.g. `sonnet`) directly to the Claude CLI rather than resolving them at config load time, keeping `load_config` pure and testable.
- [0003 — Global user-scope config and env](0003-global-user-scope-config-and-env.md): Layer `defaults → global (~/.config/pycastle/) → local (pycastle/)` for `config.py` and `.env`, so credentials and preferences can be shared across projects without per-project re-entry.
- [0004 — GitHub PAT as sole credential](0004-github-pat-as-sole-credential.md): Route all GitHub access through `GithubService` with a single `GH_TOKEN` PAT, eliminating the dual `gh`-CLI / PAT auth paths that could fail independently.
- [0005 — Dual Claude account failover](0005-dual-claude-account-failover.md): Maintain an in-memory `AccountPool` of OAuth tokens; failover to a secondary account on usage-limit rather than sleeping, collapsing all auth onto `CLAUDE_CODE_OAUTH_TOKEN` vars.
- [0006 — Session resume across container teardown](0006-session-resume-across-container-teardown.md): Resume interrupted agents via `--resume <SESSION_UUID>` (UUID derived from role + worktree path) so in-progress reasoning survives usage-limit pool sleeps; non-typed Resume failures retry once then route to Failure-Analysis (amended #640).
- [0007 — Stage completion via role-session-dir state](0007-stage-completion-via-role-dir-not-commit-prefix.md): Signal implement/review stage completion by the presence of an empty `.pycastle-session/<role>/` dir (content cleared on success), not by `RALPH:` commit-prefix scan, eliminating the unrecoverable retry loop from issue #514.
- [0008 — Centralized UsageLimitError catch at iteration boundary](0008-centralized-usagelimit-catch-at-iteration-boundary.md): Catch `UsageLimitError` once at the top of `run_iteration` rather than in each phase's dispatcher arm, preventing silent crashes when a new phase is added without boilerplate.
- [0009 — PromptRenderer with scoped placeholder tiers](0009-prompt-renderer-with-scoped-placeholder-tiers.md): A single `PromptRenderer` validates all prompt templates at startup against a two-tier placeholder system (global pool + per-scope dynamic set), so prompt edits never require test changes.
- [0010 — Per-phase session namespacing for improve mode](0010-per-phase-session-namespacing-for-improve-mode.md): Split improve phase 03 (issue-filing) onto its own Claude session namespace (`issues`) away from phases 01/02/04 (`main`), injecting the PRD body via host-side prompt args rather than inheriting the full prior transcript.
- [0011 — Recently-closed filter for GitHub list endpoint](0011-recently-closed-filter-for-github-list-endpoint.md): `GithubService` tracks a self-healing `_recently_closed` set so `get_open_issues` filters out just-closed issues during the eventual-consistency window, preventing the planner from re-picking issues the Merger just closed.
- [0012 — Prompt rendering in AgentRunner, not ContainerRunner](0012-prompt-rendering-in-agent-runner-not-container-runner.md): Move all prompt-shape resolution from `ContainerRunner.work()` into `AgentRunner._build_prompt()`, decoupling the container runner from `PromptRenderer` and improve-mode session semantics.
- [0013 — Content-addressed PreflightCache; remove skip_preflight](0013-remove-skip-preflight-route-merger-through-cache.md): Replace iteration-scoped `Deps.preflight_verdict` with a single-slot cross-iteration `PreflightCache`; remove `RunRequest.skip_preflight`, `runner.preflight()`, and `PreflightFailure`; route the Merger's post-merge check through `get_safe_sha()`.
- [0014 — Pin implement-phase SHA from planner](0014-pin-implement-sha-from-planner.md): Reintroduce `PlanReady.sha`; `run_issue` consumes the planner's SHA directly and no longer calls `get_safe_sha`, closing the duplicate-issue race in #679 when HEAD advances between planning and implement.
