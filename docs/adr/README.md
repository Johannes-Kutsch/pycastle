# Architecture Decision Records

- [0001 ‚Äî Runtime dependency installation](0001-runtime-dependency-installation.md): Install dev tools at container startup, not baked into the image.
- [0002 ‚Äî CLI-native model shorthand resolution](0002-cli-native-model-shorthand-resolution.md): Pass shorthands directly to service CLI; `load_config` stays pure.
- [0003 ‚Äî Global user-scope config and env](0003-global-user-scope-config-and-env.md): Layer `defaults ‚Üí global ‚Üí local` for `config.py` and `.env`.
- [0004 ‚Äî GitHub PAT as sole credential](0004-github-pat-as-sole-credential.md): Single `GH_TOKEN` PAT for all GitHub access.
- [0005 ‚Äî Dual Claude account failover](0005-dual-claude-account-failover.md): In-memory OAuth token pool with secondary-account failover.
- [0006 ‚Äî Session resume across container teardown](0006-session-resume-across-container-teardown.md): Resume interrupted agents via derived UUID so in-progress reasoning survives.
- [0007 ‚Äî Stage completion via role-session-dir state](0007-stage-completion-via-role-dir-not-commit-prefix.md): Empty `.pycastle-session/<role>/` dir signals done, not commit-prefix scan.
- [0008 ‚Äî Centralized UsageLimitError catch](0008-centralized-usagelimit-catch-at-iteration-boundary.md): Single `try/except` at `run_iteration` top level.
- [0009 ‚Äî PromptRenderer with scoped placeholder tiers](0009-prompt-renderer-with-scoped-placeholder-tiers.md): Startup validation of prompt templates against global + per-scope placeholders.
- [0010 ‚Äî Per-phase session namespacing for improve](0010-per-phase-session-namespacing-for-improve-mode.md): Improve phase 03 on its own session namespace (`issues`). *Phase-03-split superseded by ADR 0046; namespace mechanism + strict phase-1‚Üí2 gate + PRD-as-durable-handoff still stand.*
- [0011 ‚Äî Recently-closed filter](0011-recently-closed-filter-for-github-list-endpoint.md): Self-healing `_recently_closed` set in `GithubService`.
- [0012 ‚Äî Prompt rendering in AgentRunner](0012-prompt-rendering-in-agent-runner-not-container-runner.md): Prompt-shape resolution in `AgentRunner`, not `ContainerRunner`.
- [0013 ‚Äî Content-addressed PreflightCache](0013-remove-skip-preflight-route-merger-through-cache.md): Single-slot cross-iteration cache; Merger routes through `get_safe_sha()`.
- [0014 ‚Äî Pin implement-phase SHA from planner](0014-pin-implement-sha-from-planner.md): `run_issue` consumes planner's SHA directly.
- [0015 ‚Äî Pluggable agent services](0015-pluggable-agent-services-per-stage-fallback.md): `AgentService` at the streaming-execution seam with per-stage fallback.
- [0016 ‚Äî Three-mode implement slice](0016-three-mode-implement-slice-with-explicit-gate.md): `refactor-slice`, `behavior-slice`, `docs-slice` with explicit gates.
- [0017 ‚Äî Semantic XML tags](0017-semantic-xml-tags-for-agent-prompts.md): Fixed four-tag vocabulary for prompt structure.
- [0018 ‚Äî Persist agent drafts in role session dir](0018-persist-agent-drafts-in-role-session-dir.md): Drafts survive container teardown via preservation rule.
- [0019 ‚Äî Implicit Docker image verify](0019-implicit-image-verify-on-pycastle-run.md): `pycastle run` builds before starting containers.
- [0020 ‚Äî Codex auth via per-role `auth.json`](0020-codex-auth-via-per-role-bind-mount.md): Per-role workspace-local OAuth `auth.json` with independent refresh lineages.
- [0021 ‚Äî Shared `gh` issue-tracker fragment](0021-shared-gh-issue-tracker-fragment-in-default-prompts.md): `{{ISSUE_TRACKER}}` global placeholder from `shared/_issue-tracker.md`.
- [0022 ‚Äî Destination-scoped auto-issue gate](0022-destination-scoped-auto-issue-gate.md): `auto_file_bugs` gates upstream pycastle repo issues; default `False`.
- [0023 ‚Äî Three-bucket Claude API error handling](0023-three-bucket-claude-api-error-handling.md): 429/5xx/4xx classification at parsing layer.
- [0024 ‚Äî Per-agent CLI flag profiles](0024-per-agent-cli-flag-profiles.md): Hardcoded per-role flag profiles for token reduction.
- [0025 ‚Äî Unified status-row wrapper](0025-unified-status-row-wrapper.md): Single `status_row` CM with `kind`, `must_close`, `color_key`.
- [0026 ‚Äî Operator-actionable git failures](0026-operator-actionable-git-failures.md): Retry-then-escalate to consuming project tracker.
- [0027 ‚Äî `max_parallel` as concurrency cap](0027-max-parallel-as-concurrency-cap.md): Caps concurrent agents, not per-iteration batch size.
- [0028 ‚Äî Preserve transient worktree for Failure-Report](0028-preserve-transient-worktree-for-failure-analysis.md): Keep worktree on `AgentFailedError` for diagnostic agent.
- [0029 ‚Äî Pin agent CLI versions and codex sandbox](0029-pin-agent-cli-versions-and-fix-codex-resume-sandbox.md): Version pins and `--sandbox` only on fresh Codex runs.
- [0030 ‚Äî Live Codex status tokens](0030-live-codex-status-tokens-require-exact-prompt-side-telemetry.md): Exact prompt-side token count, not total spend.
- [0031 ‚Äî Universal agent image and stage priority chains](0031-universal-agent-image-and-stage-priority-chains.md): One image, ordered priority-list fallback chains.
- [0032 ‚Äî Host-owned conflict merge commits](0032-host-owned-conflict-merge-commits.md): Pycastle owns `git merge`/commit/ancestry validation.
- [0033 ‚Äî Host check loop](0033-host-check-loop-for-current-os-diagnostics.md): `pycastle check` runs `HOST_CHECKS` on current OS.
- [0034 ‚Äî Setup phase failures route upstream](0034-setup-phase-failures-route-upstream.md): Handled aborts routed through `auto_file_issue`.
- [0035 ‚Äî Fixed project-local layout](0035-fixed-project-local-layout.md): Fixed `pycastle/` paths; `logs_dir` stays configurable.
- [0036 ‚Äî Per-file prompt overrides](0036-per-file-prompt-overrides-without-init-scaffolding.md): Local files override bundled at same relative path.
- [0037 ‚Äî Prompt family layout](0037-prompt-family-layout.md): `coordination/`, `diagnostics/`, `improve/`, `shared/`, `work/`.
- [0038 ‚Äî Service-aware Failure-Report session dir](0038-service-aware-failure-report-session-dir.md): Failed service name through `AgentFailedError`.
- [0039 ‚Äî Shared operator-actionable credential failures](0039-shared-operator-actionable-agent-credential-failures.md): Cross-provider credential failure routing to consuming project.
- [0041 ‚Äî Managed session and diagnostic workspaces](0041-agent-session-state-only-in-managed-worktrees.md): Session state only in managed worktrees.
- [0042 ‚Äî Template-specific protocol reprompts](0042-template-specific-protocol-reprompts.md): Expected output shape shared between first prompt and reprompt.
- [0043 ‚Äî OpenCode timeout exhaustion uses unknown-reset usage-limit path](0043-opencode-timeout-exhaustion-uses-unknown-reset-usage-limit-path.md): Treat OpenCode timeout exhaustion as an unknown-reset usage limit.
- [0044 ‚Äî Host-initiated privileged worktree cleanup](0044-host-initiated-privileged-worktree-cleanup.md): Remove root-owned orphan worktrees via a host-spawned root container; never an agent capability.
- [0045 ‚Äî Failure-Report evidence from captured agent log](0045-failure-report-evidence-from-captured-agent-log.md): Copy the failed run's agent-invocation log into the worktree; never depend on provider session export.
- [0046 ‚Äî Improve slice phase resumes the PRD session](0046-improve-slice-phase-resumes-prd-session.md): Phase 03 resumes the `main` transcript instead of starting fresh in `issues`; `issues` namespace retired (supersedes the phase-03 split in ADR 0010).
